Package com.industry.analytics.flow_manipulation

Event Summary
CombinerThe Combiner Analytic takes one or more input channels, combines the unmodified event received onto a single output channel. If the optional parameter aggregatedSourceId is provided, the Combiner aggregates all incoming Data events by modifying their sourceId to the one provided. This then allows the data to be treated as a singular aggregated source. For example, if a calculation is required in which the original source of the data values are irrelevant. If the Data events type is RAW, the original sourceId is added to the generated Data events params.
DuplicatorThe Duplicator Analytic takes one input channel, and then duplicates the unmodified event received onto one or more output channels.
EventRateThe EventRate Analytic outputs the rate of arrival of events on the specified input channel. This rate can be derived based on either the correlator time or on the timestamps of the Data events.
FilterThe Filter Analytic filters the Data events passed through it based on the configuration parameters given below. Appropriate use of the sValue filters can create a stream with just float values or just string values if desired.
MapperThe Mapper Analytic will modify a field on each event in the input stream.
MergerThe Merger Analytic either adds static data or merges data from another channel onto a single data event on the output channel.
RepeaterThe Repeater Analytic takes Data events from the input channel and repeats the Data received if an update was not received within the defined time interval.
SlicerThe Slicer Analytic generates a set of 'X' new Data events for each incoming event, at 'Y' seconds apart and with value 'Z'. The value of 'Z' is defined by the behavior. The new events are generated every 'Y' seconds.
SorterIn a general use case the event ordering cannot be guaranteed. This may be because the events are batched in some way for different devices or network delays. For example, where vehicles take data readings every 5 minutes but only return the results as a batch every hour.
SuppressorThe Suppressor Analytic will suppress events from the defined input stream from appearing on the output stream name for a set duration following a trigger event. The trigger event will be passed through regardless allowing the prevention of repeated warnings, for example.
ThrottlerThe Throttler Analytic removes repeated events from the output stream.
